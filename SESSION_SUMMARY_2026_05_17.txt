PHASE 4: CODE-LEVEL GOVERNANCE RULES (NEW)
-------------------------------------------
Implemented 6 categories of mechanical rules to prevent bloat:

1. DEPENDENCY GOVERNANCE
   - Checklist before adding: why, where, lighter alt, minimum version
   - Quarterly audit: match package.json to actual imports
   - Rule: If in package.json but NOT imported → DELETE

2. DEAD CODE PREVENTION
   - No @pytest.mark.skip without expiration date (2026-06-15 format)
   - No commented-out code (delete or uncomment, don't leave hanging)
   - Quarterly scan for unused functions

3. IMPORT HYGIENE
   - Remove ALL unused imports before commit
   - Standardize datetime imports (no 14+ variations)
   - No circular imports

4. TEST SUITE HYGIENE
   - Skipped tests: must have expiration date + real reason
   - "Tested elsewhere" = DELETE (it's redundant)
   - Quarterly cleanup of expired tests

5. FRONTEND COMPONENT LIFECYCLE
   - New page = must be routed (in App.jsx + navigation)
   - Orphan pages: DELETE
   - Quarterly audit of pages/ vs actual routes

6. COMMIT MESSAGE STANDARDS
   - Removals must explain WHY
   - Example: "cleanup: remove redis (never imported, no caching impl)"
