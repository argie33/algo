ISSUE #3: Reconciliation Happens BEFORE Exits (Wrong Order)
──────────────────────────────────────────────────────────────
STATUS: CONFIRMED BUG

Evidence:

  - Phase 4: Execute exits (based on stale position data)
  - Phase 7: Final reconciliation

  Correct order should be:
  - Phase 3a: Reconcile AFTER Phase 4, before Phase 6
  - Phase 4: Execute exits
  - Phase 6: Execute entries (with accurate position count)
  - Phase 7: Final reconciliation

Impact:
  - Risk: Phase 4 exit logic based on possibly stale positions
  - Risk: Phase 6 position count might be wrong
  - Could lead to: Overleverage (thinking we have fewer positions than we do)
  - Risk Level: MEDIUM (potential to cause problems)
