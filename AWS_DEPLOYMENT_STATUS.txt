================================================================================
DATA FLOW IN AWS
================================================================================

S3 (staging) → ECS Task → PostgreSQL RDS ← API (webapp/lambda)
    ↓            ↓              ↓              ↓
  staging   Docker container  Tables      Frontend
  buckets   (loader code)      queries     dashboards

Watermarks:
  - Tracked in watermark_state table
  - Enable incremental loading (only fetch new data)
  - Speed: 5000 symbols in ~5 min vs ~4.5 hours before

Deduplication:
  - Redis Bloom filter (or in-memory fallback)
  - Pre-filters duplicates before DB write
  - Reduction: 99% fewer duplicate checks

Bulk Insert:
  - PostgreSQL COPY via S3 staging
  - Speed: 10x faster than row-by-row INSERT
  - Atomicity: all-or-nothing per batch

================================================================================
TROUBLESHOOTING
================================================================================

If GitHub Actions fails:
  1. Check .github/workflows/deploy-app-stocks.yml syntax
  2. Verify all required secrets are set
  3. Check GitHub Actions logs for specific error
  4. Common: missing AWS_ACCOUNT_ID or IAM role

If CloudFormation fails:
  1. Check AWS CloudFormation console
  2. Look for stack events/errors
  3. Verify RDS credentials in Secrets Manager
  4. Check VPC and subnet configuration

If loaders don't execute:
  1. Verify EventBridge rule is enabled
  2. Check Step Functions execution history
  3. Look at ECS task logs in CloudWatch
  4. Confirm RDS security group allows ECS subnet

If data doesn't appear:
  1. Check RDS connection from ECS task
  2. Verify database credentials
  3. Look at loader application logs
  4. Confirm watermark table exists
