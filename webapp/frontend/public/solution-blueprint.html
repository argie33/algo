4Orchestrator Phases (Detail)

PHASE 1 FAIL-CLOSED
Data Freshness
Validates SPY price, market_health_daily, and trend_template_data are from the last trading day (using MarketCalendar to skip weekends/holidays). buy_sell_daily and signal_quality_scores are observe-only — logged but never halt (loaded by EOD pipeline after Lambda fires). Halt = no trading today.
PHASE 2 FAIL-CLOSED
Circuit Breakers (13 checks)
Portfolio-level risk gates. If any trigger: skip Phases 4b, 5, 6 (no new entries), but still execute Phase 3/4 exits and Phase 7 reconciliation. Fail-closed on crashes (protective), fail-open on transient DB errors. See §9 for full list.
PHASE 3 FAIL-OPEN
Position Monitor
For each open position: fetch price, compute P&L, evaluate exit conditions, score health. Proposes: HOLD, RAISE_STOP, or EARLY_EXIT. No trades executed here — decisions queued for Phase 4.
PHASE 3b FAIL-OPEN
Market Exposure Policy
11-factor composite score (0–100) → 4 tiers: Correction (≤25%), Caution (26–50%), Pressure (51–75%), Uptrend (76–100%). Tier determines: risk multiplier, max new positions, minimum grade required. Fail-open with 0.7× default. Timeout (t4g.micro under load) is non-blocking.
PHASE 4 FAIL-OPEN
Exit Execution
Routes all exit decisions through TradeExecutor.exit_trade(): atomic DB update + Alpaca order. Handles partial exits (fraction=0.5 for T1/T2), stop-raise-only (fraction=0), and full exits. Audit-logs every action.
PHASE 4b FAIL-OPEN
Pyramid Adds
Adds to winning positions at defined extension points (+1R, +2R). Only triggered when market exposure ≥ Caution tier, position is profitable, and sector not crowded. Skipped if circuit breakers fired.
PHASE 5 FAIL-OPEN
Signal Generation
Runs BUY signal candidates through the 5-tier filter pipeline. Resolves eval_date (most recent date with full signal + market health + trend data). Ranks survivors by swing_score descending. Returns candidate list for Phase 6.
PHASE 6 FAIL-OPEN
Entry Execution
Final checks (close ≥ 60% of day's range, volume ≥ 1.25× 50d avg, sector limit ≤3, portfolio capacity). Sizes position using 6-multiplier formula. Places bracket order on Alpaca: entry + stop-loss + take-profit (OCO). Skipped if circuit breakers fired.
PHASE 7 FAIL-OPEN
Reconciliation
Pulls live Alpaca account state. Syncs positions table (fills, partial fills). Computes information coefficient (IC) and signal attribution. Runs weight optimizer. Publishes CloudWatch metrics. Generates daily performance report. 30-second timeout on all Alpaca API calls.

9Circuit Breakers (13 Checks)

All 13 checks run in Phase 2. Any triggered = no new entries (Phases 4b, 5, 6 skip). Exits and reconciliation still proceed. Fail-closed on code crashes (protective). Fail-open on transient DB/network errors (non-blocking).

CB1
Daily Loss
Today's P&L < −2% of portfolio NAV. Stops all new trades for the day.
CB2
Max Drawdown
Peak-to-current drawdown > 20% (config: halt_drawdown_pct). Full halt until recovery.
CB3
Drawdown Re-engagement
After drawdown halt: requires recovery to 50% of drawdown before resuming.
CB4
Consecutive Losses
≥ 3 consecutive losing trades (config: max_consecutive_losses). Cooldown period.
CB5
Total Risk Exposure
Aggregate portfolio risk (all open positions) exceeds configured max total risk.
CB6
Market Stage 4
Market in confirmed downtrend (Weinstein Stage 4). All new entries halted market-wide.
CB7
VIX Spike
VIX > vix_max_threshold (35 default). Extreme fear spike triggers halt.
CB8
Weekly Loss
Current week P&L < −3% of NAV. Reset Monday open.
CB9
Sector Concentration
Any single sector already at ≥ 3 positions blocks additional same-sector entries.
CB10
Intraday Market Health
SPY closed down > 2% yesterday. Short-term market stress indicator.
CB11
Win Rate Floor
Rolling 30-trade win rate < 40% (config: min_win_rate_pct). System quality degraded.
CB12
Daily Profit Cap
Today's gain > 3% of NAV. Lock in gains, no new positions needed today.
CB13
Data Freshness
Critical tables stale beyond tolerance (duplicates Phase 1 for fail-safety in Phase 2).

10Position Sizing Formula

Every position starts with a fixed risk per trade, then 6 multipliers scale it based on current market conditions. This implements Curtis Faith's R-unit framework with dynamic regime awareness. 

  CORE FORMULA
  ─────────────────────────────────────────────────────────────────────────────

  risk_dollars = NAV × base_risk_pct × dd_adj × exposure_mult × phase_mult × VIX_mult × regime_mult

  shares = risk_dollars / (entry_price − stop_loss_price)

  max_position_value = NAV × 0.08   ← 8% single position cap
  max_total_invested = NAV × 0.95   ← never fully invested

  ─────────────────────────────────────────────────────────────────────────────

  Multiplier         Formula                         Range
  ─────────────────────────────────────────────────────────────────────────────
  base_risk_pct      AlgoConfig (default 0.75%)      Fixed
  dd_adj             1.0 − (drawdown_pct / max_dd)   0.25–1.0
  exposure_mult      Market exposure tier             0.25–1.0 (Correction–Uptrend)
  phase_mult         Orchestrator run timing          0.8–1.0 (pre-close = 0.8)
  VIX_mult           1.0 − (VIX − 20)/100            0.5–1.0
  regime_mult        IBD regime: uptrend/pressure     0.5–1.0

Example Sizing Calculation

Inputs:

Portfolio NAV$100,000
base_risk_pct0.75%
Current drawdown3%
Market exposure tierPressure (0.75×)
VIX22
Entry price$150
Stop-loss$143 (−4.7%)

Calculation:

base_risk = $100,000 × 0.0075 = $750
dd_adj    = 1.0 − (3/10) = 0.70
exp_mult  = 0.75
VIX_mult  = 1.0 − (22−20)/100 = 0.98
regime    = 0.9 (pressure)

risk_dollars = $750 × 0.70 × 0.75 × 0.98 × 0.9
             = $347

shares = $347 / ($150 − $143)
       = $347 / $7 = 49 shares
position = 49 × $150 = $7,350 (7.35%)

11Exit Engine (Priority Hierarchy)

algo/algo_exit_engine.py — checks are evaluated in order; first match wins. Target exits (T1/T2) are configurable to require a pullback or trigger immediately via require_target_pullback config key.

1
Hard Stop
Price ≤ active_stop (initial or trailed). Full exit (100%). Capital preservation always wins.
2
Minervini Trend Break
Close < 50-DMA (clean break), OR close < EMA(21) on volume > 50d avg × 1.15. Full exit.
3
RS Line Break
Stock/SPY ratio falls below its own 50-day MA (config: exit_on_rs_line_break_50dma). Full exit.
4
Time Exit
Held ≥ max_hold_days (default 20). Override: O'Neil 8-week rule — if +20% in first 3 weeks, hold through 8 weeks.
5
Breakeven Stop Raise
At +1R gain: raise stop to entry price (fraction=0, no exit). Locks in break-even protection.
6
T1 Target (1.5R)
Exit 50%. Stop raised to entry. Requires pullback only if require_target_pullback=true (default false).
7
T2 Target (3R)
Exit 50% of remaining. Stop raised to T1. Requires T1 already hit. Pullback gate same as T1.
8
T3 Target (4R)
Final exit (100% of remaining). Requires T1 and T2 already hit. No pullback required at T3.
9
Chandelier / EMA Trail
At +1R: 3×ATR from highest high. After 10 days: 21-EMA trail (config: chandelier_atr_mult, switch_to_21ema_after_days).
10
TD Sequential
9-count sell setup → exit 50%. 13-count combo → full exit. Requires target_hits ≥ 1 and R ≥ 0.5.
11
First Red Day
After +2.5R gain: first big down day (−1.5%) on heavy volume (1.5× avg). Exit 50%.
12
Climax Exhaustion
30+ days held, +5R gain, +20% in last 10 days → parabolic climax distribution. Exit 50%.
13
Distribution Days
Market distribution days > max_distribution_days (4 default). Full exit (config: exit_on_distribution_day).

Tiered Exit Stop Progression

  Entry: $100       Stop: $93      (7% risk = 1R)

  At +1.5R ($110.50):  T1 → exit 50%   → raise stop to $100 (breakeven)
  At +3.0R ($121.00):  T2 → exit 50%   → raise stop to $110.50 (T1 level)
  At +4.0R ($128.00):  T3 → exit 100%  → locked in 4R on remainder

  Result: worst case after T1 = breakeven on remaining 50%
          worst case after T2 = +1.5R on remaining 25%
          best case at T3     = full position at +4R

15N / N+1 / N+2 Development Paradigm

Running paper trading while continuing development requires environment isolation. The paradigm: N = current paper algo, N+1 = in-development next version, N+2 = future live trading. 

  ┌──────────────────────────────────────────────────────────────────────────┐
  │  N   — Current Production Paper (running, stable)                        │
  │        Branch: main                                                       │
  │        Env: algo-algo-dev Lambda (existing deployment)                   │
  │        Database: algo-db (production RDS)                                │
  │        Alpaca: Paper account (ALPACA_PAPER_TRADING=true)                 │
  │        EventBridge: 4 daily runs (real schedule)                         │
  └──────────────────────────────────────────────────────────────────────────┘
                              │  (running independently)
  ┌──────────────────────────────────────────────────────────────────────────┐
  │  N+1 — Development / Next Version (in progress)                          │
  │        Branch: develop or feature/xxx                                    │
  │        Env: algo-algo-dev2 Lambda (separate deployment via tfvars)       │
  │        Database: algo-db-dev RDS instance (separate schema)              │
  │        Alpaca: Separate paper account or same paper with prefix          │
  │        EventBridge: Manual invocations or separate rule suffix "-dev2"   │
  │        Deploy: manual-invoke-loaders.yml with --function-name override   │
  └──────────────────────────────────────────────────────────────────────────┘
                              │  (backtest validates before promoting)
  ┌──────────────────────────────────────────────────────────────────────────┐
  │  N+2 — Future Live Trading                                               │
  │        Branch: main (after promotion from N+1)                           │
  │        Env: algo-algo-prod Lambda (new deployment)                       │
  │        Database: algo-db-prod (production RDS — separate from paper)     │
  │        Alpaca: Live account (ALPACA_PAPER_TRADING=false)                 │
  │        Deploy: Requires explicit terraform.tfvars change + review        │
  └──────────────────────────────────────────────────────────────────────────┘


  PROMOTION GATE (N+1 → N):
  1. Backtest N+1 on ≥ 6 months of historical data
  2. Run N+1 paper alongside N for ≥ 2 weeks
  3. Compare: win rate, expectancy, max drawdown, Sharpe
  4. If N+1 metrics ≥ N on all axes: promote to main, redeploy
  5. Keep N deployed in standby for 1 week before decomm

  ISOLATION RULES:
  • N and N+1 share the same RDS database (read-only for N+1 historical queries)
    but N+1 writes to separate tables (prefix: _dev or separate schema)
  • AlgoConfig env var: ALGO_ENV=paper|dev|prod distinguishes instances
  • DynamoDB lock key includes env: "algo-lock-{env}" prevents cross-env conflicts

Practical Steps to Set Up N+1

StepAction
1Create terraform/terraform-dev2.tfvars with different Lambda name, separate DynamoDB lock table, separate S3 prefix
2Set ALGO_ENV=dev in dev2 Lambda environment variables
3Use GitHub Actions manual-invoke-loaders.yml to backfill dev2 data tables
4Run dev2 orchestrator manually (test-orchestrator.yml with function name override)
5Compare paper results weekly — promote when ready

17Debugging Guide

Common Issues & Resolutions

SymptomRoot CauseFix
Phase 1 halts every day market_health_daily loader ran before prices loaded Check Step Functions execution order. market_health reads price_daily — must run after prices.
All signals rejected (0 candidates) Tier 2 gate, circuit breaker, or stale data Check CB log in algo_audit_log. Verify market_health_daily.distribution_days_4w and VIX values.
Orchestrator hangs at Phase 3b RDS DiskQueueDepth high → I/O contention Enable RDS Proxy (enable_rds_proxy=true in terraform.tfvars). Proxy adds connection pooling.
Lambda 500 on first request VPC cold-start: ENI provisioning takes 15–40s Reserved concurrency=1 in Terraform keeps instance warm. Retry works on warm instance.
Alpaca auth 401 Wrong API keys or wrong base URL Verify ALPACA_PAPER_TRADING, APCA_API_BASE_URL in env. Paper=paper-api.alpaca.markets.
AWS credential error locally Token expired Run: scripts/refresh-aws-credentials.ps1
T1/T2 exits never trigger Stocks blow past targets without pulling back Set require_target_pullback=false (default). Removes pullback requirement for immediate partial exits.
Swing scores all showing SQS composite Old loader stored raw SQS not weighted score Re-run load_swing_trader_scores.py — loader now computes proper 7-component weighted score.
Terraform plan fails (Invalid index) Stale state references to deleted loaders Run: terraform state rm "module.loaders.aws_ecs_task_definition.loader[\"<name>\"]"
