## Comparison: CloudFormation vs Terraform

| Feature | CloudFormation | Terraform |
|---------|---|---|
| Validation hooks | ❌ Problematic (ResourceExistenceCheck) | ✅ Built-in validation |
| State management | Hidden in AWS | Explicit + portable |
| Multi-cloud | ❌ AWS only | ✅ AWS/Azure/GCP |
| Modularity | Limited (nested stacks) | ✅ First-class modules |
| Readability | YAML (verbose) | HCL (concise) |
| Error recovery | Difficult (rollback states) | Explicit plan/apply |
| Learning curve | Steep | Moderate |

## Terraform Commands Reference

```bash
# Initialize
terraform init

# Validate syntax
terraform validate

# Format code
terraform fmt -recursive

# Plan changes
terraform plan                          # All modules
terraform plan -target=module.core      # Single module
terraform plan -out=tfplan              # Save to file

# Apply changes
terraform apply                         # All changes
terraform apply tfplan                  # Apply saved plan
terraform apply -target=module.core     # Single module
terraform apply -auto-approve           # Skip confirmation (CI/CD only)

# Inspect state
terraform state list                    # List all resources
terraform state show aws_vpc.main       # Show specific resource
terraform output                        # Display outputs
terraform output vpc_id                 # Single output

# Debugging
terraform plan -json | jq               # JSON output for parsing
terraform state list -json | jq         # JSON state
terraform console                       # Interactive REPL

# Cleanup
terraform plan -destroy                 # See what will be deleted
terraform destroy                       # Delete all resources
terraform destroy -target=module.loaders # Delete specific module
```

## State File Backup

```bash
# Backup current state (before major changes)
cp terraform.tfstate terraform.tfstate.backup

# Restore from backup
cp terraform.tfstate.backup terraform.tfstate
terraform refresh  # Sync with AWS
```

```bash
terraform workspace new prod

# List workspaces
terraform workspace list

# Switch workspaces
terraform workspace select staging

# Deploy with workspace variables
terraform apply -var-file="staging.tfvars"
```
